# Company-OS v1 Rollout Plan

## Weeks 1–2: Foundation

Validate repository structure, agent discovery, skill invocation, contracts, and approval gates. Fix configuration defects before operational use.

## Weeks 3–6: Shadow Mode

Run the scenarios in `evals/scenarios/` against realistic portfolio and venture requests. Record routing errors, role overlap, unsupported claims, missed approval gates, and unnecessary token or coordination cost.

## Weeks 7–10: Supervised Operations

Use the system for real venture discovery, product planning, and local prototypes. Keep all external, financial, legal, production, and irreversible actions behind explicit Founder approval.

## Weeks 11–13: Hardening

Repeat evaluations, consolidate overlapping roles or workflows, review governance and decision rights, and publish an evidence-backed Company-OS v1 decision.

## Exit Criteria

- All automated validation and IDE smoke scenarios pass.
- Material work is traceable from Work Order through evidence and decision.
- No observed workflow bypasses an approval gate.
- Writer conflicts are prevented or sequenced.
- The Founder accepts the remaining risks and operating cost.
