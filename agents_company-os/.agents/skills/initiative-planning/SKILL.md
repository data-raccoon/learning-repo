# Initiative Planning

1. Use `company/templates/work-order.md`.
2. Express one measurable objective and list facts separately from assumptions.
3. Choose exactly one accountable writing owner. Add other roles only for bounded read-only contribution or disjoint paths.
4. Define deliverables and acceptance criteria that can be verified with observable evidence.
5. Map every writable path to one writer. Sequence work if ownership would overlap.
6. Identify dependencies, approval gates, risk owners, and required assurance lanes.
7. Mark the Work Order `founder-approval-required` if it could cross any gate in `company/decision-rights.md`.
8. Start execution only after dependencies and ownership are unambiguous.
