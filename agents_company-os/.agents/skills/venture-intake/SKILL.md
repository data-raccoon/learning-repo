# Venture Intake

1. Create a Work Order from `company/templates/work-order.md` and assign `venture_product_lead` as writing owner.
2. State the idea as a falsifiable thesis. Separate known evidence from assumptions.
3. Use `market_intelligence` and `customer_research` as read-only contributors when evidence is missing.
4. Complete `company/templates/venture-charter.md`, including lifecycle stage, KPI tree, budget ceiling, and observable kill criteria.
5. Define the smallest experiments that can disprove the highest-risk assumptions before product build-out.
6. Save the draft under `ventures/<venture-slug>/` based on `ventures/_template/`.
7. Invoke `$council-decision` before moving beyond discovery or committing material capital.
