# Weekly Operating Review

1. Set a cutoff and gather current Venture Charters, Work Orders, KPI Trees, Council Decisions, risk registers, and quality results.
2. Have `operations_knowledge` consolidate status without rewriting source records.
3. Report each venture's lifecycle stage, KPI movement, delivered evidence, blocked work, capital or capacity concern, and next decision.
4. Flag stale decisions, overdue reviews, unowned risks, missing evidence, and approval requests awaiting the Founder.
5. Ask `finance_capital_allocator` for read-only input when runway, budget, or allocation changed materially.
6. Produce a short portfolio summary plus proposed next-week priorities. Do not convert proposals into approval.
7. Record approved priorities as Work Orders and preserve the review as an immutable dated artifact.
