# Operating Model

## Flow of Work

1. Intake: frame the request as a Work Order.
2. Route: select one accountable owner and any read-only contributors.
3. Decide: invoke the Agent Council when a strategic threshold is crossed.
4. Execute: keep write ownership exclusive and changes scoped.
5. Assure: obtain independent quality, security, privacy, and operational evidence as applicable.
6. Approve: request Founder approval when an approval gate applies.
7. Record: update decisions, risks, KPIs, and reusable knowledge.

## Agent Council

The Council provides five independent lenses: portfolio fit, capital efficiency, product evidence, technical feasibility, and legal/risk exposure. The Chief of Staff consolidates without suppressing dissent. The Founder approves, rejects, defers, or requests more evidence.

## Venture Lifecycle

`idea -> intake -> discovery -> incubating -> validating -> scaling -> sustaining | paused | stopped`

Each transition requires evidence, an accountable owner, a review date, and explicit kill criteria.
